## Phase 1: Multi-Instrument Price Model + TUI Display

### Overview
Replace the 2-instrument random walk with an 8-instrument correlated GBM + mean-reversion price model. Expand engine arrays from [2] to [8]. All 8 instruments produce realistic L2Books and candles visible in the TUI.

### Changes

#### `build.zig` — Add new module imports to desk build targets
- Add imports to `desk_main_mod` and `desk_test_mod` for: `"basis"` (basis_mod), `"funding_arb"` (funding_arb_mod), `"twap"` (twap_mod), `"vpin"` (vpin_mod), `"tca"` (tca_mod), `"eod"` (eod_mod), `"reconciliation"` (reconciliation_mod)
- Also add these to `headless_main_mod` and `headless_test_mod`
- Note: The existing modules `orderbook`, `oms`, `order_types`, `positions`, `pre_trade`, `bar_aggregator` are already imported

#### `trading/desk/synthetic.zig` — Rewrite to multi-instrument correlated model
- Replace `SyntheticFeed` struct: change `books: [2]L2Book` → `books: [8]L2Book`, `base_prices: [2]i64` → `base_prices: [8]i64`
- Add `instruments: [8]InstrumentConfig` field and `correlation_groups: [8]u8` field
- Add `InstrumentConfig` struct:
  ```zig
  pub const InstrumentConfig = struct {
      symbol: [32]u8,
      symbol_len: u8,
      base_price: i64,
      volatility: i64,       // tick-size multiplier for random step amplitude
      mean_reversion_strength: i64, // pull-back factor (0 = pure random walk)
      drift: i64,            // per-tick drift component
      correlation_group: u8, // spot/perp pairs share a group
  };
  ```
- Define 8 instrument configs as a comptime array:
  - Group 0 (BTC): BTC-USD (spot, base=5_000_000_000_000, vol=500_000, mr=100_000, drift=0), BTC-USD-PERP (perp, base=5_002_000_000_000, vol=50_000, mr=50_000, drift=0), BTC-USD-20231229 (dated, base=5_010_000_000_000, vol=50_000, mr=30_000, drift=0)
  - Group 1 (ETH): ETH-USD (spot, base=300_000_000_000, vol=300_000, mr=80_000, drift=0), ETH-USD-PERP (perp, base=300_100_000_000, vol=30_000, mr=40_000, drift=0), SOL-USD-PERP (perp, base=10_000_000_000, vol=20_000, mr=40_000, drift=0)
  - Group 2 (SOL): SOL-USD (spot, base=10_000_000_000, vol=200_000, mr=60_000, drift=0)
  - Group 3 (ADA): ADA-USD (spot, base=50_000_000, vol=5_000, mr=20_000, drift=0)
- Rewrite `init()`: initialize all 8 books with `L2Book.init(allocator, DEPTH)`, populate each with its config base_price
- Rewrite `tick()`:
  1. Generate one random step per correlation group (groups 0,1,2,3)
  2. For each instrument: `delta = group_step * config.volatility / base_volatility + independent_noise`
  3. Apply mean reversion: `pull = config.mean_reversion_strength * (config.base_price - current_price) / config.base_price`
  4. `new_price = current_price + delta + pull + config.drift`
  5. Clamp to prevent negative prices: `new_price = @max(new_price, config.base_price / 10)`
  6. Update book levels around new base as before (1-3 random level updates per tick, full repopulate every 20 ticks)
- Add `pub fn computeFundingRate(spot_mid: i64, perp_mid: i64) i64` — returns `k * (perp_mid - spot_mid) * SCALE / spot_mid` clamped to ±1% (k=100, integer basis points scaled to i64)
- Update `getBook()` to work with index 0..7
- Add `pub fn getSymbol(self: *const SyntheticFeed, index: usize) []const u8` — returns symbol slice from InstrumentConfig
- Update existing test; add test for 8-instrument correlation (seed=42, verify BTC spot/perp mid prices within 2% of each other after 100 ticks)

#### `trading/desk/engine.zig` — Expand from 2 to 8 instruments
- Change `INSTRUMENTS` array from 2 to 8 entries: `"BTC-USD", "ETH-USD", "SOL-USD", "ADA-USD", "BTC-USD-PERP", "ETH-USD-PERP", "SOL-USD-PERP", "BTC-USD-20231229"`
- Change `candle_aggs: [2]BarAggregator` → `candle_aggs: [8]BarAggregator`
- Update `init()`: initialize 8 BarAggregators
- Update `run()` loop: iterate `0..8` instead of `0..2` for book snapshots and candle aggregation
- Update `StatusUpdate` push: change `instrument_count: 2` → `instrument_count: 8`

#### `trading/desk/messages.zig` — Expand StatusUpdate
- Add `instrument_count: u8` field already exists, no change needed for Phase 1
- Ensure `StatusUpdate.instrument_count` is already a u8 (confirmed: line 62)

### Edge cases
- Negative price after mean reversion overshoot: clamp to `base_price / 10` minimum
- Empty book (0 bids or 0 asks) after populateBook: cannot happen since populateBook always writes DEPTH=20 levels
- Correlation group with single instrument (groups 2, 3): independent_noise dominates, which is fine

### Breaking changes
- None — internal struct changes only; TUI receives more orderbook_snapshot events but the format is unchanged

## Phase 2: Real Order Pipeline (Risk, Matching, Positions, P&L)

### Overview
Create a matching engine that fills orders against synthetic books. Replace risk stubs with real PreTradeRisk. Replace SimplePosition with real PositionManager. Orders submitted in TUI go through risk → OMS → matching → position → P&L.

### Changes

#### `trading/desk/matching_engine.zig` — NEW
- Create `MatchingEngine` struct:
  ```zig
  pub const MatchingEngine = struct {
      resting_orders: [256]RestingOrder,
      resting_count: usize,
  };
  ```
- `RestingOrder` struct: `order_id: u64, instrument_idx: u8, side: Side, price: i64, remaining_qty: i64, active: bool`
- `pub fn init() MatchingEngine` — zero-initialize resting array
- `pub fn processOrder(self: *MatchingEngine, order_id: u64, instrument_idx: u8, side: Side, price: ?i64, quantity: i64, order_type: OrderType, book: *const L2Book) FillResult`:
  - Market orders: fill immediately at BBO. Walk price levels if quantity exceeds top-of-book. Produce FillInfo with volume-weighted avg price.
  - Limit orders crossing BBO: fill the crossing portion, rest the remainder.
  - Limit orders not crossing: store as resting order.
  - Return `FillResult` struct: `fills: [16]FillInfo, fill_count: u8, rested: bool, rested_qty: i64`
- `pub fn checkRestingOrders(self: *MatchingEngine, books: *const [8]L2Book) RestingFillResult`:
  - Iterate resting orders. For each: check if current BBO has crossed the limit price.
  - Buy resting: fills if `bestAsk().price <= resting.price`
  - Sell resting: fills if `bestBid().price >= resting.price`
  - Return array of `RestingFill`: `order_id: u64, fill: FillInfo`
  - Mark filled resting orders as `active = false`
- `FillResult` struct: `fills: [16]FillInfo, fill_count: u8, rested: bool, rested_qty: i64`
- `RestingFill` struct: `order_id: u64, fill_qty: i64, fill_price: i64`
- `RestingFillResult` struct: `fills: [64]RestingFill, fill_count: u8`
- Import `Side` from `order_types`, `L2Book` from `orderbook`, `FillInfo` from `oms`
- Add tests: market buy fills at ask, market sell fills at bid, limit order rests then fills on price cross, partial fill when book liquidity insufficient

#### `trading/desk/engine.zig` — Wire real risk, matching, and positions
- Add imports: `const pre_trade = @import("pre_trade")`, `const PreTradeRisk = pre_trade.PreTradeRisk`, `const positions_mod = @import("positions")`, `const PositionManager = positions_mod.PositionManager`, `const MatchingEngine = @import("matching_engine.zig").MatchingEngine`
- Remove `riskValidateStub`, `storeAppendStub`, `dummy_risk`, `dummy_store`, `SimplePosition`, `updatePosition()`
- Add to `Engine` struct:
  - `risk: PreTradeRisk` — replaces dummy_risk
  - `pos_manager: PositionManager` — replaces SimplePosition array + position_count
  - `matching: MatchingEngine` — new
- Add `fn riskValidateWrapper(risk_ptr: *anyopaque, order: *const Order) bool`:
  - Cast `risk_ptr` to `*PreTradeRisk`
  - Call `risk.validate(order)` → return `true` if `.passed`, `false` if `.rejected`
- Add `fn storeAppendNoop(store: *anyopaque, data: []const u8) anyerror!u64` — keep noop store (no persistence needed in demo)
- Update `init()`:
  - Initialize `PreTradeRisk.init(allocator, .{ .max_order_size = 1_000_000_00000000, .max_notional = 50_000_000_00000000, .max_position = 5_000_000_00000000, .max_order_rate = 100, .price_band_pct = 0.10, .dedup_window_ms = 1000 })`
  - Initialize `PositionManager.init(allocator, .{ .cost_basis_method = .average_cost, .base_currency = "USD" })`
  - Initialize `MatchingEngine.init()`
  - Pass `@ptrCast(&self.risk)` and `storeAppendNoop` to `OrderManager.init()`
  - Note: risk is a pointer stored in Engine, so self pointer must be stable. Use `@ptrCast` on the `*PreTradeRisk` field.
- Update `deinit()`: add `self.risk.deinit()`, `self.pos_manager.deinit()`
- Update `run()` loop — after `feed.tick()`, add:
  1. Call `self.matching.checkRestingOrders(&self.feed.books)` — for each fill, call `self.oms.onExecution(fill.order_id, .fill, .{ .fill_qty = fill.fill_qty, .fill_price = fill.fill_price })` and feed fill to position manager
  2. Update ref prices on risk for each instrument: `self.risk.setReferencePrice(symbol, mid_price)`
- Rewrite `handleOrderRequest()`:
  1. Build `Order` struct as before
  2. `self.oms.submitOrder(order)` — if error (risk rejection), push rejected OrderUpdate to TUI, return
  3. Determine `instrument_idx` from instrument string
  4. Call `self.matching.processOrder(order_id, instrument_idx, side, price, quantity, order_type, book)`
  5. For each fill in result: call `self.oms.onExecution(order_id, exec_type, fill_info)` where exec_type is `.fill` if fully filled or `.partial_fill`
  6. Feed each fill to `self.pos_manager.onFill(positions_mod.Fill{ ... })`
  7. Push `order_update` event with real status and filled_qty
- Update position_update push in run() loop:
  - Iterate known instruments, look up position in `self.pos_manager` via key
  - Call `self.pos_manager.unrealizedPnl(key, mark_price)` where mark_price = current mid
  - Push `PositionUpdate` with real quantity, avg_cost, unrealized_pnl, realized_pnl
- Remove `positions: [16]SimplePosition` and `position_count: usize` fields

### Edge cases
- Order submitted for unknown instrument (not in INSTRUMENTS array): reject with error OrderUpdate
- Market order on empty book (no asks for buy, no bids for sell): fill at base_price as fallback
- Position reversal (long → short via oversized sell): PositionManager handles this natively
- Risk self-pointer stability: Engine is heap-allocated via the caller; pointer to risk field is stable for OMS lifetime

### Breaking changes
- None — EngineEvent variants unchanged; position_update now has real values instead of zeros

## Phase 3: Strategy + Algo Automation

### Overview
Wire basis and funding arb strategies into the engine loop. When a strategy emits a Signal, the engine creates a TWAP algo that submits child orders through the full pipeline (risk → OMS → matching → positions). User observes strategy activity in TUI status updates.

### Changes

#### `trading/desk/engine.zig` — Add strategy and algo fields and tick logic
- Add imports: `const basis = @import("basis")`, `const funding_arb = @import("funding_arb")`, `const twap_mod_algo = @import("twap")`, `const TwapAlgo = twap_mod_algo.TwapAlgo`, `const TwapParams = twap_mod_algo.TwapParams`, `const ChildOrder = twap_mod_algo.ChildOrder`
- Add to `Engine` struct:
  ```
  basis_strategy: basis.BasisStrategy,
  funding_arb_strategy: funding_arb.FundingArbStrategy,
  active_algos: [16]ActiveAlgo,
  active_algo_count: usize,
  ```
- Define `ActiveAlgo` struct:
  ```zig
  const ActiveAlgo = struct {
      twap: TwapAlgo,
      parent_order_id: u64,
      instrument_idx: u8,
      side: twap_mod_algo.Side,
      arrival_price: i64,
      active: bool,
  };
  ```
- Update `init()`:
  - Initialize `basis.BasisStrategy.init(allocator, .{ .entry_threshold_bps = 50.0, .exit_threshold_bps = 10.0, .max_position = 100_000_00000000, .instrument_spot = "BTC-USD", .instrument_futures = "BTC-USD-PERP", .days_to_expiry = 90 })`
  - Initialize `funding_arb.FundingArbStrategy.init(allocator, .{ .min_rate_bps = 5.0, .max_position = 100_000_00000000, .instrument_spot = "ETH-USD", .instrument_perp = "ETH-USD-PERP" })`
  - Zero-initialize `active_algos`, `active_algo_count = 0`
- Update `deinit()`: deinit strategies
- Add `fn runStrategies(self: *Engine) void`:
  1. Get BTC spot book (idx 0) and BTC-USD-PERP book (idx 4): call `self.basis_strategy.onMarketData(&books[0], &books[4])`
  2. If Signal returned with direction `.enter_long_basis` or `.enter_short_basis`:
     - Create parent order via OMS for the spot leg
     - Create TWAP with params: `total_qty = signal.spot_qty`, `start_time = current_tick_ns`, `end_time = current_tick_ns + 60_000_000_000` (60s), `num_slices = 10`, `instrument = "BTC-USD"`, `side = .buy/.sell based on signal`, `jitter_pct = 0.1`
     - Store as ActiveAlgo with arrival_price = current mid
     - Create second TWAP for futures leg (opposite side)
  3. Compute funding rate via `SyntheticFeed.computeFundingRate(eth_spot_mid, eth_perp_mid)`
  4. Call `self.funding_arb_strategy.onFundingRate(rate_as_f64, current_tick_ns)`
  5. If Signal returned: create TWAP pair for spot + perp legs as above
  6. Call `self.funding_arb_strategy.onMarketData(&books[1], &books[5])` for convergence monitoring
- Add `fn runAlgos(self: *Engine) void`:
  1. For each active algo: call `twap.nextSlice(current_tick_ns)`
  2. If ChildOrder returned: submit through pipeline:
     - Build Order from ChildOrder fields
     - `self.oms.submitOrder(order)` → if ok, `self.matching.processOrder(...)` → fills → `self.oms.onExecution(...)` → `self.pos_manager.onFill(...)`
  3. If `twap.isComplete()`: mark ActiveAlgo as `active = false`
- Update `run()` loop — after feed.tick() and before command drain:
  1. Call `self.runStrategies()`
  2. Call `self.runAlgos()`

#### `trading/desk/messages.zig` — Add strategy state to StatusUpdate
- Add field `strategy_state: [64]u8` and `strategy_state_len: u8` to `StatusUpdate`
- This carries a human-readable string like `"basis:long ETH_arb:flat"` for TUI display

#### `trading/desk/engine.zig` — Update StatusUpdate push
- Format strategy state string from `basis_strategy.state` and `funding_arb_strategy.position` into `StatusUpdate.strategy_state`

### Edge cases
- Strategy emits signal but active_algo_count == 16: skip signal, log nothing (bounded array prevents overflow)
- TWAP nextSlice returns null (not yet time for next slice): no action, continue
- Strategy signal with zero quantity: skip (strategies guarantee nonzero qty per design)
- Funding rate division by zero (spot_mid == 0): impossible since price is clamped to base_price/10 minimum in Phase 1

### Breaking changes
- `StatusUpdate` struct gains new fields (`strategy_state`, `strategy_state_len`) — TUI panels that read this struct must handle the new fields. Since the TUI reads via union variant and StatusUpdate fields are additive, this is backward-compatible at the binary level (fixed-size ring buffer items).

## Phase 4: Analytics + Post-Trade

### Overview
Wire VPIN per-instrument toxicity scoring, TCA reports on algo completion, and periodic EOD P&L snapshots. Push analytics data to TUI via enriched StatusUpdate and new EngineEvent variants.

### Changes

#### `trading/desk/engine.zig` — Add analytics and post-trade fields
- Add imports: `const vpin_mod_analytics = @import("vpin")`, `const VpinCalculator = vpin_mod_analytics.VpinCalculator`, `const tca_mod_analytics = @import("tca")`, `const TcaEngine = tca_mod_analytics.TcaEngine`, `const eod_mod_analytics = @import("eod")`, `const EodProcessor = eod_mod_analytics.EodProcessor`
- Add to `Engine` struct:
  ```
  vpin: [8]VpinCalculator,
  tca_engine: TcaEngine,
  eod_processor: EodProcessor,
  tca_pending: [16]TcaPending,
  tca_pending_count: usize,
  eod_tick_counter: u64,
  ```
- Define `TcaPending` struct:
  ```zig
  const TcaPending = struct {
      algo_idx: usize,
      fills: [128]tca_mod_analytics.Execution,
      fill_count: usize,
      benchmark: tca_mod_analytics.Benchmark,
      active: bool,
  };
  ```
- Update `init()`:
  - Initialize 8 VpinCalculators: `VpinCalculator.init(allocator, 100_000_00000000, 20)` (bucket_size=100K units, 20 buckets)
  - Initialize `TcaEngine.init(allocator)`
  - Initialize `EodProcessor.init(allocator)`
  - Zero-initialize `tca_pending`, `eod_tick_counter = 0`
- Update `deinit()`: deinit all 8 VpinCalculators, TcaEngine, EodProcessor

#### `trading/desk/engine.zig` — Add analytics tick logic
- Add `fn runVpin(self: *Engine) void`:
  1. For each instrument 0..7: get mid_price from book
  2. Call `self.vpin[i].onTrade(mid_price, 1, .buy)` (use trade direction from last price move or .buy as default)
  3. Store latest VPIN scores for StatusUpdate
- Add `fn completeTca(self: *Engine, pending_idx: usize) void`:
  1. Get the TcaPending entry
  2. Call `self.tca_engine.analyze(fills_slice, pending.benchmark)`
  3. Push TCA report as EngineEvent to TUI
  4. Mark pending as `active = false`
- Add `fn runEod(self: *Engine) void`:
  1. Increment `eod_tick_counter`
  2. Every 6000 ticks (= 10 minutes at 100ms, simulating EOD):
     - Build `EodPositionView` array from position manager state
     - Build `Mark` array from current mid prices
     - Call `self.eod_processor.computeDailyPnl(positions, marks)`
     - Push `DailyPnlReport` as EngineEvent to TUI
     - Reset counter
- Update `runAlgos()`: when a TWAP completes, call `completeTca()` for its pending entry
- Update `runAlgos()`: when creating fills from child orders, record each fill in the corresponding TcaPending entry
- When creating a new ActiveAlgo (in `runStrategies`), also create a TcaPending with arrival_price benchmark

#### `trading/desk/engine.zig` — Update run() loop ordering
- After feed.tick():
  1. `self.runVpin()` — feed trade data to VPIN
  2. `self.runStrategies()` — check for strategy signals
  3. `self.runAlgos()` — advance TWAPs, may complete and trigger TCA
  4. `self.runEod()` — periodic EOD check
  5. Push enriched StatusUpdate with VPIN scores

#### `trading/desk/messages.zig` — Add new event types and extend StatusUpdate
- Add `vpin_scores: [8]i64` and `vpin_valid: [8]bool` fields to `StatusUpdate` (i64 = VPIN * 10000 for fixed-point; bool = whether score is available yet)
- Add new EngineEvent variants:
  ```zig
  tca_report: TcaReportEvent,
  eod_report: EodReportEvent,
  ```
- Define `TcaReportEvent` struct (fixed-size for ring buffer):
  ```zig
  pub const TcaReportEvent = struct {
      instrument: InstrumentId,
      is_cost_bps: i64,       // scaled by 100
      fill_rate_pct: u8,      // 0-100
      market_impact_bps: i64, // scaled by 100
  };
  ```
- Define `EodReportEvent` struct:
  ```zig
  pub const EodReportEvent = struct {
      realized_pnl: i64,
      unrealized_pnl: i64,
      total_pnl: i64,
      tick: u64,
  };
  ```

### Edge cases
- VPIN returns null (first bucket not yet full): push `vpin_valid[i] = false` in StatusUpdate
- TCA with zero fills (TWAP completed but all slices were risk-rejected): produce report with fill_rate=0, all costs=0
- EOD with no positions: produce report with all zeros (valid state)
- TcaPending array full (16 concurrent algos): skip TCA tracking for new algo (bounded, non-critical)
- f64→i64 conversion for VPIN scores: multiply by 10000 and truncate; VPIN is 0.0–1.0 so result fits i64

### Breaking changes
- `EngineEvent` union gains two new variants (`tca_report`, `eod_report`) — TUI switch statements must add handlers (can be no-ops initially). This is the only breaking change; existing TUI code must add `else` or explicit match arms.
- `StatusUpdate` struct size increases — ring buffer items are already union-sized so this is fine as long as StatusUpdate doesn't become the largest variant. Verify OrderbookSnapshot (20*2 PriceLevels + InstrumentId ≈ 672 bytes) is still largest.
