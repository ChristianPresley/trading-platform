## Phase 4: zcov Coverage Tool + Integration

### Overview
Build the pure-Zig coverage tool at `test/zcov/` and integrate it into the build system and VS Code tasks.

### Changes

#### `test/zcov/main.zig` — NEW
- Entry point for coverage tool. `pub fn main() !void`:
  1. Parse CLI args for optional module filter and output options
  2. Compile test binaries with coverage instrumentation (invoke `zig build test` with `-fcode-coverage` or equivalent nightly flag)
  3. Run test binaries to generate coverage data in `.zig-cache/`
  4. Call `coverage.collectCoverage()` to read coverage data
  5. Call `report.printReport()` to produce terminal output
- Uses `std.process.Child` to invoke zig build and run test binaries
- Uses `std.heap.GeneralPurposeAllocator` for memory

#### `test/zcov/coverage.zig` — NEW
- `pub const CoverageData = struct { source_file: []const u8, lines_hit: []bool, total_lines: u32, covered_lines: u32 }`
- `pub fn collectCoverage(allocator: std.mem.Allocator, test_binary_path: []const u8) ![]CoverageData`:
  1. Open the test binary ELF file using `std.elf`
  2. Parse DWARF debug info using `std.debug.Dwarf` to map PCs to source file:line
  3. Read coverage instrumentation data (PC bitfield from `-fcode-coverage` in `.zig-cache/`)
  4. Cross-reference covered PCs with DWARF line info to build per-file line coverage
  5. Return array of `CoverageData` structs
- Helper: `fn mapPcToSource(dwarf: *std.debug.Dwarf, pc: usize) !?struct { file: []const u8, line: u32 }`
- Helper: `fn readCoverageData(allocator: std.mem.Allocator, path: []const u8) ![]usize` — reads the raw PC list from coverage output

#### `test/zcov/report.zig` — NEW
- `pub const ModuleSummary = struct { name: []const u8, total_lines: u32, covered_lines: u32, pub fn percentage(self: ModuleSummary) f64 }`
- `pub fn printReport(writer: std.io.AnyWriter, data: []const CoverageData) !void`:
  1. Group `CoverageData` entries by module (sdk/core, sdk/domain, sdk/protocol, exchanges/kraken, trading/desk, trading/analytics, trading/strategies) based on source file path prefix
  2. Compute `ModuleSummary` for each module
  3. Print header with total coverage percentage
  4. Print per-module summary table with colored bars (green=covered, red=uncovered using ANSI escape codes)
  5. Print per-file detail for files below threshold or on request
- Helper: `fn classifyModule(source_path: []const u8) []const u8` — maps file path to module name
- Helper: `fn printBar(writer: std.io.AnyWriter, pct: f64, width: u32) !void` — renders colored progress bar

#### `build.zig` — Add zcov build step
- After the existing test steps (~line 1297), add:
  - `const zcov_mod = b.createModule(.{ .root_source_file = b.path("test/zcov/main.zig") })` — zcov module
  - `const zcov_exe = b.addExecutable(.{ .name = "zcov", .root_module = zcov_mod, .target = target, .optimize = optimize })` — zcov executable
  - `b.installArtifact(zcov_exe)` — install to zig-out/bin/
  - `const zcov_run = b.addRunArtifact(zcov_exe)` — run step
  - `const zcov_step = b.step("zcov", "Run coverage analysis")` — named step
  - `zcov_step.dependOn(&zcov_run.step)` — wire up
- Update comment at top of build.zig to document the new step

### Edge cases
- Coverage data format: Zig nightly's `-fcode-coverage` (or equivalent) may produce data in an undocumented format. The tool must inspect `.zig-cache/` after a coverage-instrumented run and adapt to what's actually produced. If no built-in coverage instrumentation exists, fall back to DWARF-based PC sampling via `std.debug.Coverage` if available, or document the limitation.
- Large binaries: the ELF/DWARF parsing must handle the full platform binary (~29K LOC). Use streaming reads, not full file loads.
- Files outside module boundaries: source files in the root or unexpected paths should be grouped under "other" in the report.
