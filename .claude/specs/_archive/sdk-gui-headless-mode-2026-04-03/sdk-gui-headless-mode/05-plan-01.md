## Phase 1: Engine Module Extraction + Candle Pipeline

### Overview
Extract the engine into a shared module importable by multiple executables. Rename the current `desk` executable to `desk-tui`. Add `CandleUpdate` to the message types and wire `BarAggregator` into the engine to produce candle events from synthetic feed price ticks. All existing behavior is preserved ‚Äî this is refactoring + candle data plumbing.

### Changes

#### `trading/desk/messages.zig` ‚Äî add CandleUpdate type and EngineEvent variant
- Add `CandleUpdate` struct after `StatusUpdate` (line 64):
  ```zig
  pub const CandleUpdate = struct {
      instrument: InstrumentId,
      open: i64,
      high: i64,
      low: i64,
      close: i64,
      volume: i64,
      timestamp: u64,
  ```

- Add `candle_update: CandleUpdate` variant to `EngineEvent` union (after `status` variant, line 71)

#### `trading/desk/engine.zig` ‚Äî wire BarAggregator, produce candle events
- Add import: `const bar_agg = @import("bar_aggregator");` and `const BarAggregator = bar_agg.BarAggregator;`
- Add import: `const CandleUpdate = msg.CandleUpdate;`
- Add fields to `Engine` struct:
  - `candle_aggs: [2]BarAggregator` ‚Äî one per instrument
- In `Engine.init()`: initialize `candle_aggs` with `BarAggregator.init(60_000_000_000)` (1-minute bars in nanoseconds) for each instrument
- In `Engine.run()` main loop, after `self.feed.tick()` and before pushing orderbook snapshots: for each instrument, get BBO midpoint from the book (`(best_bid + best_ask) / 2`), call `self.candle_aggs[i].onTrade(midpoint, 1, timestamp_ns)` where `timestamp_ns` is `self.tick * 100_000_000` (tick count √ó 100ms in ns).   - `var candle_counts: [2]usize = .{ 0, 0 };`

- Ring buffer full when pushing candle_update: silently dropped (consistent with existing push pattern ‚Äî `_ = self.to_tui.push(...)`)

## Phase 2: Headless Executable

### Overview
Create `desk-headless` binary that runs the engine without any terminal dependency. Exposes a `HeadlessDesk` struct with `push(UserCommand)` / `pop() ?EngineEvent` API for programmatic control. Enables integration testing and future algo trading.

### Changes

#### `trading/desk/headless_main.zig` ‚Äî NEW
- New file containing the headless executable entry point and public API
- `HeadlessDesk` struct with fields:
  - `to_engine: *SpscRingBuffer(UserCommand)` ‚Äî commands into engine
  - `from_engine: *SpscRingBuffer(EngineEvent)` ‚Äî events from engine
  - `engine: Engine`
  - `engine_thread: std.Thread`
  - `allocator: std.mem.Allocator`
- `pub fn init(allocator: Allocator) !HeadlessDesk`:
  - Allocate two `SpscRingBuffer` instances (capacity 256)
  - Create `Engine` with those buffers
  - Spawn engine thread via `std.Thread.spawn(.{}, Engine.run, .{&self.engine})`
  - Return populated struct
- `pub fn push(self: *HeadlessDesk, cmd: UserCommand) bool`: forward to `self.to_engine.push(cmd)`
- `pub fn pop(self: *HeadlessDesk) ?EngineEvent`: forward to `self.from_engine.pop()`
- `pub fn shutdown(self: *HeadlessDesk) void`:
  - Push `UserCommand.quit`
  - Spin-drain `from_engine` until `EngineEvent.shutdown_ack` received (with timeout counter)
  - Join engine thread
  - Free ring buffers and deinit engine
- `pub fn main() !void`: minimal main that creates `HeadlessDesk`, runs for a few ticks, prints event count, shuts down (demo entry point)
- Inline tests:
  - `test "headless_init_shutdown"`: init HeadlessDesk, immediately shutdown, verify clean join
  - `test "headless_push_pop"`: init, sleep 200ms for engine to produce events, pop and verify at least one `EngineEvent` received, then shutdown
  - `test "headless_quit_ack"`: init, push quit, spin-pop until shutdown_ack, verify received

#### `trading/desk/engine.zig` ‚Äî make ring buffer field types accessible
- No structural changes needed ‚Äî `Engine.init()` already accepts ring buffer pointers as parameters. HeadlessDesk passes its own ring buffers.

#### `build.zig` ‚Äî add desk-headless executable and build steps
- Create `headless_main_mod`: `b.createModule(.{ .root_source_file = b.path("trading/desk/headless_main.zig"), .target = target, .optimize = optimize })`
- Add same imports as `desk_main_mod`: orderbook, oms, order_types, positions, pre_trade, memory, time, ring_buffer, bar_aggregator
- Create `desk_headless_exe`: `b.addExecutable(.{ .name = "desk-headless", .root_module = headless_main_mod })`
- Add build steps: `build-desk-headless`, `run-desk-headless`, `test-desk-headless`
- The headless module needs to import desk engine code. Since `headless_main.zig` is in the same directory as `engine.zig`, it can use `@import("engine.zig")` directly (same as `main.zig` does).

### Edge cases
- Engine thread not producing events fast enough: `pop()` returns null, caller retries ‚Äî non-blocking by design
- Double shutdown: guard with a `stopped: bool` flag in HeadlessDesk to prevent double-join
- Shutdown timeout: if engine doesn't ack within 1000 drain iterations with 1ms sleep between, force-stop via `engine.requestStop()` then join

### Breaking changes
- None

### Test checkpoint
- command: `zig build test-desk-headless`
- expected-output: "(none ‚Äî exit code 0 is sufficient)"

## Phase 5: Sparklines, Depth Visualization + Animations

### Overview
Add BBO sparklines and depth bars to the orderbook panel, fill/reject flash highlights to the orders panel, and status message fade animation. Final visual polish phase.

### Changes

#### `trading/desk/panels/orderbook_panel.zig` ‚Äî add sparkline and depth bars

- Add `history` parameter: ring buffer of recent BBO midpoints for sparkline rendering

  - Maps quantity/max_quantity ratio to bar width using `‚ñë‚ñí‚ñì‚ñà` block characters
