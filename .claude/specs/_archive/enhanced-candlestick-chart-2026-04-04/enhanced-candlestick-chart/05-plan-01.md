## Phase 1: Half-Block Candlestick Rendering + Theme

### Overview
Create shared chart primitives module with half-block character constants and sub-cell scaling, add 4 new theme colors, and rewrite candlestick rendering to use 2x vertical resolution with distinct 3-col bodies and 1-col dimmed wicks. Fix Y-axis labels to show 2 decimal places.

### Changes

#### `trading/desk/panels/chart_primitives.zig` — NEW
- New shared module for chart rendering primitives and indicator types
- Define `HALF_BLOCK_CHARS: [8][3]u8` — the 8 lower-block characters (`▁▂▃▄▅▆▇█`, U+2581..U+2588), same values as `orderbook_panel.zig:27-36` but public
- Define `UPPER_HALF: [3]u8` — `▀` (U+2580) = `{ 0xe2, 0x96, 0x80 }`
- Define `FULL_BLOCK: [3]u8` — `█` (U+2588) = `{ 0xe2, 0x96, 0x88 }`
- Define `SubCell` struct:
  ```zig
  pub const SubCell = struct {
      row: u16,   // terminal row
      half: u1,   // 0 = top half, 1 = bottom half
  };
  ```
- Implement `pub fn scaleYSub(price: i64, min_price: i64, max_price: i64, height: u16) SubCell`:
  - Doubles effective resolution: maps price to `(height * 2 - 1)` sub-cell positions
  - Formula: `sub_pos = (height * 2 - 1) - ((price - min_price) * (height * 2 - 1) / range)`
  - Returns `SubCell{ .row = @intCast(sub_pos / 2), .half = @intCast(sub_pos % 2) }`
  - Guard: if range == 0, return `SubCell{ .row = height / 2, .half = 0 }`
  - Guard: if height == 0, return `SubCell{ .row = 0, .half = 0 }`
- Define `Indicator` struct (used in Phase 4, but defined here for the framework):
  ```zig
  pub const Indicator = struct {
      name: [16]u8,
      name_len: u8,
      period: u16,
      color: Rgb,
      compute: *const fn (candles: []const CandleUpdate, index: usize, period: u16) ?i64,
  };
  ```
  - Requires importing `Rgb` from `../theme.zig` and `CandleUpdate` from `../messages.zig`
- Implement `pub fn smaCompute(candles: []const CandleUpdate, index: usize, period: u16) ?i64`:
  - Returns null if `index + 1 < period` (insufficient data)
  - Sums `candles[index + 1 - period .. index + 1]` close prices, divides by period
  - Uses i64 arithmetic throughout (prices are fixed-point with 8 decimal places)
- Add tests:
  - `test "scaleYSub_boundaries"` — verify max price maps to `SubCell{0, 0}`, min price maps to bottom, midpoint maps to middle
  - `test "scaleYSub_flat_range"` — verify range==0 returns height/2
  - `test "scaleYSub_zero_height"` — verify height==0 returns {0, 0}
  - `test "smaCompute_exact"` — 5 candles with known close prices, verify SMA(3) = exact expected value
  - `test "smaCompute_insufficient"` — verify returns null when index < period - 1
  - `test "half_block_chars_length"` — verify HALF_BLOCK_CHARS has 8 entries, each 3 bytes

#### `trading/desk/panels/chart_panel.zig` — rewrite candle rendering with half-block precision
- Add import: `const primitives = @import("chart_primitives.zig");`
- Replace `scaleY` function (lines 14-23) — keep it for backward compat in tests, but add:
  ```zig
  const SubCell = primitives.SubCell;
  const scaleYSub = primitives.scaleYSub;
  ```
- Rewrite `drawCandle()` (lines 27-76) to use half-block rendering:
  - Compute `high_sub`, `low_sub`, `body_top_sub`, `body_bot_sub` using `scaleYSub()`
  - **Body rendering (3 columns)**: iterate rows from `body_top_sub.row` to `body_bot_sub.row`
    - At `body_top_sub.row`: if `body_top_sub.half == 1`, use `▄` (lower-half) with fg=candle color, bg=default; else use `█`
    - At `body_bot_sub.row`: if `body_bot_sub.half == 0`, use `▀` (upper-half) with fg=candle color, bg=default; else use `█`
    - Middle rows: use `█` (full block)
    - Write across all 3 columns: `inner_x + x`, `inner_x + x + 1`, `inner_x + x + 2`
  - **Wick rendering (center column only)**: use dimmed color variant (50% brightness of candle color)
    - Compute dimmed color: `Rgb{ .r = color.r / 2, .g = color.g / 2, .b = color.b / 2 }`
    - Upper wick: rows from `high_sub.row` to `body_top_sub.row - 1` — draw `│` in center column
    - Lower wick: rows from `body_bot_sub.row + 1` to `low_sub.row` — draw `│` in center column
    - Wick tips: at `high_sub.row` if `high_sub.half == 1`, use `▄` for half-cell precision; at `low_sub.row` if `low_sub.half == 0`, use `▀`
  - Use `renderer.writeBgColor()` + `renderer.writeColor()` for fg/bg compositing at half-block boundaries
- Rewrite Y-axis labels (lines 130-138) to show 2 decimal places:
  - Change format from `"{d}"` to include fractional part:
    ```zig
    const whole = @divTrunc(price, 100_000_000);
    const frac = @abs(@rem(price, 100_000_000)) / 1_000_000; // 2 decimal places
    const str = std.fmt.bufPrint(&pbuf, "{d}.{d:0>2}", .{ whole, frac });
    ```
  - Apply to all 3 labels (max, mid, min)

### Edge cases
- **Flat market (y_max == y_min)**: `scaleYSub` guard returns height/2 — body renders as single half-block at midpoint
- **Single-row chart (height == 1)**: all sub-cells map to row 0, half 0 or 1 — renders minimal candle
- **Body top == body bottom (doji candle)**: body_top_sub == body_bot_sub — render single half-block character for body
- **Wick extends beyond body by < 1 full row**: half-block tip handles sub-cell precision correctly

### Breaking changes
- `Theme` struct gains 4 new fields — any code constructing `Theme` literals must add them. Only 3 literal constructions exist (dark, light, classic_green in `theme.zig`) — all updated in this phase.

#### `trading/desk/panels/chart_panel.zig` — viewport-aware rendering
- Add helper functions:
  ```zig
  fn visibleCandles(panel_width: u16, candle_width: u8) usize {
      return panel_width / candle_width;
  }

  fn clampViewport(offset: usize, visible: usize, total: usize) usize {
      if (total <= visible) return 0;
      return @min(offset, total - visible);
  }
  ```
- Update visible candle slicing in `draw()` to use viewport:
  ```zig
  const max_vis = visibleCandles(inner_w, candle_width);
  const total = candles.len;
  // Auto-follow: if viewport_offset == 0, show newest candles (right edge)
  const effective_offset = if (viewport_offset == 0) 
      (if (total > max_vis) total - max_vis else 0)
  else 
      clampViewport(total -| viewport_offset -| max_vis, max_vis, total);
  const end = @min(effective_offset + max_vis, total);
  const visible = candles[effective_offset..end];
  ```
  - `viewport_offset == 0` means "auto-follow" (show newest)
  - `viewport_offset > 0` means "scrolled N candles from the right edge"
- Add tests:
  - `test "visibleCandles_calculation"` — verify: width=38, candle_width=3 → 12; width=38, candle_width=1 → 38; width=38, candle_width=5 → 7
  - `test "clampViewport_bounds"` — verify: offset beyond total clamps; offset within range passes through; total <= visible returns 0
  - `test "clampViewport_auto_follow"` — verify offset=0 with total > visible returns total-visible

### Edge cases
- **Fewer candles than visible**: viewport_offset forced to 0, all candles shown
- **Zoom changes while scrolled**: visible count changes, clampViewport adjusts offset to prevent overrun
- **Instrument switch while scrolled**: each instrument has independent viewport_offset, so switching preserves scroll position per instrument
